## 📌 TL;DR（执行摘要）

- **核心目标**：做一个微信小程序，让情侣通过自然语言对话完成记账。输入"午饭32用微信"→AI自动解析→确认卡片→一键入账，全程≤15秒。
- **关键决策**：技术栈选定微信小程序原生 + 腾讯云开发Cloudbase + DeepSeek API；内测期成本 <¥1/月；按8周排期上架。
- **差异化卖点**：①一句话记一天的批量解析；②花谁的钱进谁的账（账户驱动可见性）；③极简确认零学习成本。
- **下一步**：W1开始Sprint 1（登录+账户管理），W2末进行AI可用性Go/No-Go决策。

## 2. 用户故事（5 个场景）

### US1：日常快速记账
> 作为用户，我想要输入"午饭32用微信"，以便立即生成确认卡片并一键入账。

**验收标准**：
- 输入后≤3秒返回解析结果（分类=餐饮、金额=32、账户=微信）
- 确认入账后记录保存、首页刷新，全流程≤15秒

### US2：批量记账（核心差异化）
> 作为用户，我想要一次性输入多条消费，以便AI逐条解析后批量入账。

**验收标准**：
- 输入"午饭32，地铁4块，超市买水3.5，都用支付宝"→返回3条独立确认卡片
- 一键入账后3条同时保存，账户余额同步扣减

### US3：共同账户记账
> 作为情侣用户，我想要使用共同账户记账，以便双方都能看到这笔记录。

**验收标准**：
- 选小荷包账户记账→标记为共同可见，双方可见
- 对方不可删除自己创建的共同记录

### US4：预算超支提醒
> 作为用户，我想要看到月度预算进度条并在即将超支时收到提醒。

**验收标准**：
- 首页实时进度条（已花/预算）
- 当月消费达80%→进度条变红并弹窗提醒

### US5：首次设置+情侣绑定
> 作为新用户，我想要在3分钟内完成初始设置并绑定伴侣。

**验收标准**：
- 微信登录→创建账户→填余额→生成邀请码，全流程≤3分钟
- 对方输入6位邀请码→绑定成功，共同账户自动出现

### SWOT 分析

| 优势（S） | 劣势（W） |
|-----------|-----------|
| 批量解析是市场空白，仅AI记账助手部分支持 | 无语音记账（一期不做） |
| 账户驱动可见性是真正的产品创新 | 品牌知名度为零 |
| 情侣场景原生设计，竞品无专为情侣设计 | 个人主体限制商业化速度 |
| DeepSeek低成本优势 | |
| **机会（O）** | **威胁（T）** |
| 微信小程序生态内无真正AI原生情侣记账产品 | 咔皮记账（商汤）若推出小程序版将降维打击 |
| 批量补记场景需求强，目前无人做透 | 微信/支付宝可能内置AI记账截流 |
| DeepSeek成本优势可长期免费运营 | AI记账助手与记账喵喵先发优势明显 |

### 3个差异化卖点

1. **「一句话记一天」**：批量解析做到极致，一次性输入所有消费
2. **「花谁的钱，进谁的账」**：按支付账户自动决定可见范围，简单优雅
3. **「极简确认，零学习成本」**：AI解析+确认卡片，只需扫一眼点确认

## 7. 关键流程图

### 流程 A：日常记账（"午饭32用微信"→入账）

```
启动小程序 → 微信自动登录
    ↓
输入框输入"午饭32用微信"
    ↓
云函数调用 DeepSeek API 解析
    ↓ 返回 JSON
{ type: "支出", category: "餐饮", amount: 32, account: "微信" }
    ↓
前端渲染确认卡片（可编辑金额/分类/账户/删除）
    ↓
用户点击「确认入账」
    ↓
写入 Cloudbase transactions 集合 + 更新 accounts 余额
    ↓
首页刷新 → 校验预算阈值（达80%则弹窗提醒）
```

### 流程 B：批量记账

```
输入"午饭32，地铁4块，超市买水3.5，都用支付宝"
    ↓
AI 返回 3 条结构化数据
    ↓
前端渲染 3 张独立确认卡片（可逐条编辑/删除）
    ↓
底部显示「一键入账（3条）」
    ↓
批量写入 3 条 transaction + 统一更新余额
    ↓
首页刷新
```

### 流程 C：首次设置+情侣绑定

```
启动 → 微信一键授权登录
    ↓
引导页：创建首个账户（名称+类型+期初余额）
    ↓
提示"和另一半一起记账？"
    ↓
生成 6 位邀请码 → 分享给伴侣
    ↓
对方打开小程序 → 注册 → 输入邀请码
    ↓
绑定成功 → 共同账户出现 → 引导首次记账
```

## 8. Non-goals（明确不做什么）

| 不做项 | 原因 |
|--------|------|
| 语音输入/OCR识别发票 | 一期聚焦打字→确认最短路径 |
| 自定义分类图标/颜色 | v1默认图标，降低设计成本 |
| 多币种/汇率换算 | 复杂度高，需求小 |
| Web端/APP端 | 聚焦小程序 |
| 社交功能（记账圈/社区） | 与核心场景无关 |
| 理财建议/AI财务分析 | 超出记账范畴 |
| 周期记账/自动扣款提醒 | 二期考虑 |
| 第三方记账数据导入 | 一期先做原生体验 |
| 家庭多成员（>2人） | 一期聚焦情侣双人场景 |
